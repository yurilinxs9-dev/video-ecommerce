# Story 4.2 — Build Script: esbuild gerando widget.js

**Epic:** 4 — Widget Embeddable
**Status:** Draft
**Prioridade:** Must
**Estimativa:** 2h

## User Story

Como desenvolvedor,
Quero um script de build que gera `public/widget.js` standalone,
Para que o widget seja servido como arquivo único embeddable.

---

- [ ] **AC-1:** `scripts/build-widget.js` usa esbuild para bundle
- [ ] **AC-2:** Entry: `src/widget/mount.ts`
- [ ] **AC-3:** Output: `public/widget.js`

- [ ] **AC-5:** Bundle inclui CSS inline via style inject
- [ ] **AC-6:** Minificado e otimizado para produção
- [ ] **AC-7:** `package.json` tem script: `"build:widget": "node scripts/build-widget.js"`

## Notas Técnicas

```javascript
// scripts/build-widget.js
const esbuild = require('esbuild');
const { sassPlugin } = require('esbuild-sass-plugin'); // se usar Sass

esbuild.build({
  entryPoints: ['src/widget/mount.ts'],
  bundle: true,
  minify: true,
  platform: 'browser',
  target: ['es2020'],
  outfile: 'public/widget.js',
  define: {
    'process.env.NODE_ENV': '"production"',
  },
  // CSS: inject via plugin ou style-inject pattern
  plugins: [
    // CSS modules → inject style tag
  ],
  external: [
    // NÃO externalizar React (deve ser incluído no bundle)
  ],
}).then(() => {
  console.log('Widget bundle built successfully!');
  // Verificar tamanho gzipped
}).catch(() => process.exit(1));
```

### Nota sobre Tailwind no Widget
- Tailwind não deve estar no bundle do widget (muito grande)
- Usar inline styles ou CSS mínimo necessário
- Alternativa: PostCSS + PurgeCSS para extrair apenas classes usadas
